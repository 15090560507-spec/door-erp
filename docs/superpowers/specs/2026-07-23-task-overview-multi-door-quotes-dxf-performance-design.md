# 任务总览、多门报价与 DXF 性能优化设计

日期：2026-07-23

## 目标

在不改变现有 CAD 绘图算法、不破坏旧报价数据的前提下，完成以下调整：

1. 将“汇总看板”和“后台管理”合并为“任务总览”。
2. 报价默认保持单门，需要时可增加多樘门。
3. 主门品名按“门型 材料 正面款式 反面款式”生成。
4. 增加默认关闭的“记忆报价”，将有效报价明细写入配件库。
5. 减少 DXF 预览和下载的重复生成，并补充耗时定位。

## 任务总览

- 顶部移除“汇总看板”，将原“后台管理”入口改为“任务总览”。
- 所有登录角色都可以进入任务总览。
- 普通角色只看到任务统计、筛选和权限范围内的任务。
- 超级管理员额外看到账号管理、下拉选项等管理功能。
- 后端权限保持现有安全边界，不能仅依赖前端隐藏。

## 多门报价

### 交互

- 新建报价默认创建一个门组。
- 门组可以增加、复制和删除，最后一个门组不能删除。
- 每个门组可以独立关联图纸、选择报价模式、填写门套单价并增减明细。
- 客户名称、项目名称、日期和报价说明属于整张报价单。
- 每个门组有小计，整张报价单有总计。

### 数据兼容

- 新数据增加 `doorGroups`。
- 旧数据只有 `items` 时，在读取阶段自动转换成一个门组。
- 保存时保留顶层兼容字段，避免旧导出和历史报价立即失效。
- 明细行不再固定为 8 行；空行仅用于界面编辑，不写入有效报价数据。

### 主门品名

关联图纸生成报价时，首行按以下字段顺序组合：

`门型 材料 正面款式 反面款式`

空值自动跳过，字段之间使用空格，不使用加号。

### 导出

- Excel、JPG 和 PDF 都按门组顺序连续输出。
- 每个门组先输出主门，再输出该门配件。
- 内容超过一页时自动续页，仍属于同一张报价单。
- 每组显示小计，最后显示整单合计。
- Excel 明细金额和合计继续使用公式，保留后期编辑联动。

## 记忆报价

- “记忆报价”默认关闭，不持久化开启状态。
- 开启后，在保存或导出前批量处理当前报价的有效明细。
- 仅保存名称非空、单位非空、单价大于 0 的明细。
- 主门归类为“门类组合”；其他明细优先沿用来源配件分类，再按名称推断合页、锁具、包装或配件。
- 使用“名称 + 分类”作为更新键，同名同分类更新单位和单价，不重复新增。
- 批量入库失败时不阻断报价保存或导出，但要向用户显示明确提示。

## DXF 性能

当前本机剖析显示：

- 模板解析约占 63%。
- 完整 DXF 序列化约占 28%。
- 预览和下载会对相同参数重复生成。

第一阶段采用低风险优化：

1. 在后端记录模板读取、绘图、序列化和总耗时。
2. 对规范化后的绘图参数计算稳定哈希。
3. 将生成的 DXF 放入有容量和有效期限制的缓存。
4. 预览和下载通过同一缓存读取结果。
5. 参数变化时自动失效，不复用旧图。

暂不缓存可变的 `ezdxf` 文档对象，避免请求之间发生图元污染。

## 验证

- 所有角色可进入任务总览，管理设置仅超级管理员可见。
- 旧报价可正常打开为单门报价。
- 单门报价流程保持可用。
- 多门报价可以增加、删除门组和明细，并正确汇总。
- 导出超过 8 行时不丢数据、不遮挡、不破坏公式。
- 记忆关闭时配件库不变；开启时有效明细正确新增或更新。
- 相同参数先预览再下载时，DXF 绘图只执行一次。
- 后端测试、前端构建和关键导出测试全部通过。
